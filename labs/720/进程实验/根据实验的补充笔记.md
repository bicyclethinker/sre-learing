# Linux 进程与作业管理——核心查漏笔记

## 一、先分清三层

### 1. Shell 作业

常用命令：

```bash
jobs -l
fg %1
bg %1
```

特点：

- 只管理当前 Shell 创建并记录的任务
- `%1` 是 job 编号
- 换一个终端后，`jobs` 看不到原终端的任务

------

### 2. 系统进程

常用命令：

```bash
ps
pstree
kill
```

核心字段：

```text
PID     进程编号
PPID    父进程编号
STAT    进程状态
TTY     关联终端
COMMAND 完整命令
```

推荐查看命令：

```bash
ps -p PID -o pid,ppid,stat,tty,command
```

查看多个 PID：

```bash
ps -p 1668,1669,1670 -o pid,ppid,stat,command
```

------

### 3. systemd 服务

常用命令：

```bash
systemctl status httpd
systemctl start httpd
systemctl stop httpd
systemctl restart httpd
journalctl -u httpd
```

systemd 服务应优先使用 `systemctl` 管理，不要直接：

```bash
pkill httpd
```

------

# 二、前台、后台和暂停

## 前台运行

```bash
ping baidu.com
```

占用当前终端。

## 后台运行

```bash
ping baidu.com &
```

Shell 可以继续输入命令。

注意：

```text
后台运行不等于脱离终端。
```

## 暂停任务

前台任务中按：

```text
Ctrl+Z
```

此时进程没有退出，`STAT` 通常包含：

```text
T
```

## 后台继续

```bash
bg %1
```

## 拉回前台

```bash
fg %1
```

每次变化后检查：

```bash
jobs -l
ps -p PID -o pid,ppid,stat,tty,command
```

------

# 三、为什么 ps 能看到，jobs 看不到

`jobs` 查看的是：

```text
当前 Shell 的作业表
```

`ps` 查看的是：

```text
整个系统的进程表
```

所以重新登录后：

- 原来的任务可能仍然运行
- 新 Bash 没有旧 Bash 的作业记录
- `ps` 能看到
- `jobs` 看不到

这属于正常现象。

------

# 四、正常终止进程

优先使用：

```bash
kill PID
```

等价于：

```bash
kill -15 PID
```

终止后验证：

```bash
ps -p PID
```

不要习惯性使用：

```bash
kill -9 PID
```

`kill -9` 是强制终止，进程没有机会正常清理。

也不要随便使用：

```bash
pkill ping
```

因为可能杀死多个同名进程。

------

# 五、重定向

```bash
>     覆盖文件
>>    追加文件
```

例如：

```bash
ping baidu.com > baidu.log 2>&1 &
```

其中：

```text
2>&1
```

表示把错误输出也写入同一个日志文件。

注意：

```bash
touch null
```

创建的是普通文件。

真正用于丢弃输出的是：

```bash
/dev/null
```

例如：

```bash
ping baidu.com > /dev/null 2>&1 &
```

------

# 六、判断服务不能只看进程

不能只执行：

```bash
ps aux | grep httpd
```

完整检查：

```bash
systemctl is-active httpd
ss -lntp | grep ':80'
curl -I http://127.0.0.1
journalctl -u httpd -n 20
```

分别检查：

```text
服务状态
端口监听
实际访问
服务日志
```

------

# 七、做实验的固定流程

```text
1. 查看操作前状态
2. 只执行一个操作
3. 再次查看状态
4. 比较发生了什么变化
5. 最后清理测试进程
```

核心检查命令：

```bash
jobs -l
ps -p PID -o pid,ppid,stat,tty,command
pstree -sp PID
```

------

# 八、必须记住的八句话

1. job 编号不是 PID。
2. `jobs` 只看当前 Shell 的任务。
3. `ps` 可以查看系统中的进程。
4. `Ctrl+Z` 是暂停，不是退出。
5. `bg` 后台继续，`fg` 拉回前台。
6. 后台运行不等于脱离终端。
7. 正常终止优先使用 `kill -15`。
8. systemd 服务优先使用 `systemctl` 管理。
