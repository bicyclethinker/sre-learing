## 练习一：基础复现——作业、PID与进程状态

### 背景

在当前 Bash 中启动三个安全的长时间任务，分别模拟：

- 前台任务
- 普通后台任务
- 输出重定向的后台任务

### 要求

1. 查看当前 Bash 的 PID。
2. 启动三个任务。
3. 记录每个任务的：
   - job编号
   - PID
   - PPID
   - STAT
   - COMMAND
4. 将一个前台任务暂停。
5. 把它放到后台继续运行。
6. 再把它拉回前台。
7. 使用正常终止信号结束一个任务。
8. 验证它是否真正退出。
9. 最后清理所有测试任务。

### 限制

- 不允许使用 `kill -9`。
- 不允许操作 PID 1。
- 不允许使用 sshd 作为实验对象。
- 每次状态变化后必须同时使用 Shell 作业工具和系统进程工具检查。

### 验收标准

能够解释：

```
job编号、PID、PPID、STAT、前台、后台、停止、运行
```

之间的关系。

## 练习二：综合场景——测试服务与后台监控

### 背景

服务器上运行一个专门用于实验的 Web 服务。你需要管理服务，并启动一个独立的后台监控任务。

### 要求

1. 确认测试服务是否已经安装。
2. 查看服务当前状态。
3. 启动服务。
4. 找出：
   - 服务主进程PID
   - 父进程
   - 完整命令行
   - CPU和内存使用情况
5. 重启服务，比较重启前后的PID。
6. 启动一个后台监控任务，定期把服务状态写入日志文件。
7. 让监控任务在退出当前终端后仍能继续运行。
8. 重新登录后找到该监控进程。
9. 停止监控进程。
10. 查看服务相关日志，确认操作时间和结果。

### 限制

- 不得使用 sshd 作为测试服务。
- 不得直接使用强制信号。
- 不得只看 `ps | grep` 就认定服务正常。
- 不允许让多个后台任务覆盖同一个日志文件。

### 验收标准

能够区分：

```
systemd服务单元
服务主进程
Shell后台作业
脱离终端运行的进程
```

## 练习三：故障排查——为什么 ps 能看到，jobs 却看不到

### 故障现象

管理员发现一个长期运行的测试进程：

- 使用系统进程工具可以看到。
- 当前终端执行 `jobs` 却没有显示。
- 使用 `fg %1` 提示没有对应作业。
- 该进程的父进程可能已经发生变化。
- 不确定它是普通后台任务、nohup任务，还是systemd服务。

### 排查要求

按顺序完成：

1. 确认当前 Shell PID。
2. 找到目标进程 PID。
3. 查看目标进程 PPID。
4. 查看完整进程树。
5. 检查当前 Shell 的作业表。
6. 判断目标进程是否由当前 Shell 创建。
7. 判断它是否仍与当前终端关联。
8. 判断它是否由 systemd 管理。
9. 写出为什么 `jobs` 看不到它。
10. 使用合适方式终止或管理该进程。

### 限制

- 不允许直接 `kill -9`。
- 不允许把“jobs没有显示”解释成“进程不存在”。
- 不允许用进程名盲目批量终止。
- 修改前必须先写出至少两个假设。
